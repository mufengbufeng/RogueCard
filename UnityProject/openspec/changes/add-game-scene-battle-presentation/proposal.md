## Why

当前卡牌 Rogue 最小闭环已经具备关卡和波次相关配置基础，但还缺少承载局内表现的 Unity 场景、GameView 入口和战斗舞台预制体约定。现在先定义局内场景表现骨架，可以让后续怪物阵型、卡牌 UI、波次推进和战斗动效在同一个稳定容器内逐步接入。

## What Changes

- 新增 `Game` 场景作为局内运行场景，承载局内相机、舞台根节点和 UI 打开入口。
- 新增 `GameView`/`GameController` 局内窗口，用于承载玩家状态、手牌区、目标选择表现和结束回合等后续 UI。
- 新增战斗舞台预制体结构约定，采用 2D 世界空间表现背景、怪物、前景和世界特效。
- 新增背景分层滚动表现的最小能力，用于波次推进或前进过渡时播放远景、中景、近景滚动动画。
- 新增 3x3 怪物槽位表现能力，为怪物前后层级、排序、缩放、选中锚点和血条锚点提供固定挂载点。
- 明确局内表现采用“2D 战斗舞台 + UI 操作层”的混合方案，不使用全 UI 实现怪物和背景。
- 不在本变更中实现完整战斗规则、卡牌效果、怪物 AI、奖励结算或正式美术资源。

## Capabilities

### New Capabilities

- `game-scene-battle-presentation`: 定义 Game 场景、GameView、2D 战斗舞台、背景滚动和怪物槽位表现的最小可实施能力。

### Modified Capabilities

无。

## Impact

- 影响 Unity 场景资源、局内 UI Prefab、战斗舞台 Prefab 和 HotFix 层局内 UI/表现脚本。
- 需要在 HotFix 游戏流程或后续关卡流程中具备打开 `GameView` 的入口，但本变更只建立最小场景与窗口骨架。
- 不修改 Luban 配置表结构，不引入新的第三方依赖，不改变现有 Runtime/HotFix 分层约束。
